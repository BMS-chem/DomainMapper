DomainMapper — Overview

Browser tool for building maps of protein architecture.

Toolbar

  • Undo / Redo (Ctrl+Z / Ctrl+Y, or Ctrl+Shift+Z)
  • Load example: display a worked example
  • Import UniProt: enter an accession to fetch domains, motifs, regions, and binding/active sites from UniProt and customisably add them as a new protein
  • Import JSON: load a previous .json file, replaces current diagram
  • File name: base name for all exports
  • Export JSON: full data + all settings that can be re-imported later
  • Export image: choose SVG or PNG
  • Reset: clear canvas

General formatting

  • Format text for protein names, domain names, annotations
  • Axis length: sets absolute horizontal width of longest displayed protein; other proteins are always scaled in relation to the longest one.

Protein list

  • + Add: new protein
  • Checkbox: show/hide in diagram
  • Click name to select for editing
  • ↑ / ↓: reorder
  • ⧉: duplicate
  • ✕: delete

Protein formatting

  • Details: Name, length, horizontal offset, and coordinate axis formatting
  • Domains: enter domains, toggle start/end residue visibility, format domain box appearance, and annotate protein domains
  • Regions: label regions, groups of domains, or areas of the protein (e.g. disordered region)
  • Motifs: label motifs of interest (e.g. a binding site within a domain)
  • Annotations: label sites of interest (e.g. a residue, mutation, or interaction partner)

Overlapping region labels, motif tags, and annotations stack vertically to try and avoid overlaps (up to a point).

Privacy

DomainMapper uses Umami Analytics to collect anonymous usage statistics such as page visits and feature usage. Umami does not use cookies, does not collect personally identifiable information, and does not track you across other websites. No protein, domain, or other diagram data you enter into DomainMapper is ever recorded or sent to Umami.

Import from UniProt

Please note - UniProt import is intended to save time setting up protein annotations, but manual editing (especially the text of domain names, regions, etc) will probably be needed to produce a clean and tidy schematic.

If you would like additional types of UniProt annotations to be imported, please submit a suggestion.

Sorry - this feature isn't available when the tool is run locally. Please visit the live version hosted at https://bms-chem.github.io/DomainMapper/ to use UniProt import.
